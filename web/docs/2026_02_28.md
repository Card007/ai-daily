### 20. 求求了，别再用 Passkeys 来加密用户数据

[Please, please, please stop using passkeys for encrypting user data](https://simonwillison.net/2026/Feb/27/passkeys/#atom-everything) — **simonwillison.net** · 02-27 22:49 · ⭐ 23/30

> 把 passkeys（基于 WebAuthn/FIDO2 的无密码凭证）直接用于加密用户数据，会把“账号可用性”风险放大成“数据永久不可恢复”的灾难。关键问题在于用户经常丢失 passkeys（换手机、重装系统、同步失败、丢失安全密钥等），而很多人并不理解“一旦用它做客户端加密，丢了就再也解不开”。文章引用 Tim Cappalli 的警告，强调 passkeys 适合做身份验证（authentication），不适合作为长期稳定的数据加密根密钥（encryption key material），尤其是在没有强恢复机制的产品里。正确方向是将 passkeys 用于登录与密钥解封装/授权流程，同时为加密引入可恢复的密钥管理（例如多设备密钥备份、恢复密钥、分层密钥与轮换策略），避免把用户置于不可逆的锁死状态。结论是：passkeys 让登录更安全不等于让加密更可靠，把二者混用会在支持与数据恢复层面制造大量不可承受的事故。
