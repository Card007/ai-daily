🥉 **Please, please, please stop using passkeys for encrypting user data**

[Please, please, please stop using passkeys for encrypting user data](https://simonwillison.net/2026/Feb/27/passkeys/#atom-everything) — simonwillison.net · 02-27 22:49 · 🔒 Security

> Using passkeys (including passkey-related PRF-style schemes) as the sole basis for encrypting user data creates a high-risk recovery trap. Passkeys are frequently lost due to device churn, account changes, and sync failures, and users generally do not understand that losing a passkey can mean permanent loss of access to data encrypted under it. The warning is that “secure by design” authentication credentials do not automatically make good long-term encryption keys, because availability and recoverability requirements differ from login requirements. A safer design typically separates authentication from data-key management by adding explicit recovery mechanisms (for example, escrowed keys, user-managed backup keys, or multi-party recovery flows) instead of making the passkey the single point of failure. Takeaway: do not turn a usability failure (lost passkeys) into irreversible data loss by tying encryption directly to passkeys without a robust recovery model.
