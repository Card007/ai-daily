### 6. 如果我的对话框里有一个非按钮控件却使用了 IDCANCEL 这个控件 ID，会发生哪些糟糕的事？

[What sort of horrible things happen if my dialog has a non-button with the control ID of IDCANCEL?](https://devblogs.microsoft.com/oldnewthing/20260302-53/?p=112098) — **devblogs.microsoft.com/oldnewthing** · 03-02 20:55 · ⭐ 17/30

> Windows 对话框把 IDCANCEL 当作“取消/按 Esc 关闭”的特殊 ID 来处理，因此它不仅是一个普通控件标识，还会参与默认的键盘与关闭逻辑。若某个非按钮控件（例如静态文本、编辑框等）误用了 IDCANCEL，按下 Esc 或触发取消路径时，系统仍可能向该 ID 发送对“按钮点击”语义的命令/通知。结果是窗口过程会收到看起来不匹配的通知码或消息组合（比如像按钮被点击一样的信号），但对应控件类型根本不支持这些语义。这样的“语义错配”会让事件路由、日志与调试都变得迷惑，并可能导致错误的分支被执行。核心观点是：IDCANCEL 这类保留语义的控件 ID 不应复用在非按钮控件上，否则会引入难以解释的消息与行为。

🏷️ Windows, Win32, dialogs, IDCANCEL
