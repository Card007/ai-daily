### 6. What sort of horrible things happen if my dialog has a non-button with the control ID of IDCANCEL?

[What sort of horrible things happen if my dialog has a non-button with the control ID of IDCANCEL?](https://devblogs.microsoft.com/oldnewthing/20260302-53/?p=112098) — **devblogs.microsoft.com/oldnewthing** · 03-02 20:55 · ⭐ 16/30

> Using the special dialog control ID IDCANCEL on something that is not actually a push button causes the dialog manager to treat that control as the Cancel target anyway. Standard dialog behaviors (notably the Esc key and default cancel routing) will generate IDCANCEL-related command traffic even though the control cannot meaningfully behave like a button. That mismatch leads to WM_COMMAND notifications that look like “button clicks” or cancel actions, which can confuse your message-handling logic and make debugging feel nonsensical. The issue is not that Windows is random, but that the dialog framework keys off the numeric ID and applies button semantics regardless of the underlying control class. The practical takeaway is to reserve IDCANCEL for a real Cancel button (or at least a control intended to act like one) and give every other control a non-reserved ID.

### 21. Differential equation with a small delay

[Differential equation with a small delay](https://www.johndcook.com/blog/2026/03/02/small-delay/) — **johndcook.com** · 03-02 16:08 · ⭐ 15/30

> Delay-differential equations (DDEs) depend on past state values (e.g., y(t-τ)) and can behave very differently from ordinary differential equations, even when the delay τ is small. A common technical approach for “small delay” is to expand the delayed term with a Taylor series, such as y(t-τ) ≈ y(t) − τy'(t) + O(τ^2), turning the DDE into an approximate ODE (often with a small parameter) that is easier to analyze. This approximation clarifies how the delay perturbs stability and dynamics without requiring full history-function machinery or specialized DDE solvers. It also highlights the limits: the method is only trustworthy when τ is small relative to the system’s natural timescale and when higher-order terms remain negligible. The takeaway is that small delays can often be handled with perturbation-style approximations, but you must explicitly track the assumptions and error terms.
