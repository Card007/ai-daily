[An AI Odyssey, Part 2: Prompting Peril](https://www.johndcook.com/blog/2026/03/04/an-ai-odyssey-part-2-prompting-peril/) — johndcook.com · 03-04 14:04 · 🤖 AI / ML

> 在使用 OpenAI API 的项目协作中，一个关键疑问是：能否通过调整 API 调用方式，让模型“多想一会儿”从而提高回答准确率。作者提出这个设想后，同事立即去问 ChatGPT 是否存在这样的办法，暴露出把模型当作“权威使用手册”的常见倾向。文章强调，关于“通过提示或参数让推理更充分就更准确”的直觉并不可靠，模型对自身能力与机制的解释也可能误导实际工程决策。相比在对话里寻求肯定，更应通过可复现的实验、对照测试与指标验证来判断改动是否真的提升效果。核心观点是：提示工程的危险不在于不会写提示，而在于把未经验证的模型建议当成事实并据此改系统。

💡 **为什么值得读**: 用一个真实协作场景提醒你别被“让模型多推理=更准”这种听起来合理的说法带偏，特别适合做 API 集成与评测设计的人快速校正方法论。
