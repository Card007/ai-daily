### 9. 在到达主消息循环之前就被分发的 Posted 消息之谜

[The mystery of the posted message that was dispatched before reaching the main message loop](https://devblogs.microsoft.com/oldnewthing/20260305-00/?p=112114) — **devblogs.microsoft.com/oldnewthing** · 03-05 15:00 · ⭐ 20/30

> 问题出在：明明用 PostMessage 投递到线程消息队列的消息，为什么还没进入主消息循环（GetMessage/PeekMessage）就已经被 DispatchMessage“处理”了。关键点是 Win32 的消息分发并不只发生在你写的那段主循环里，只要某段代码把消息取出来并调用 DispatchMessage，它就会被发送到窗口过程执行。常见触发场景包括在其他“消息泵”里（例如自写的 PeekMessage 循环、等待对象时的消息泵、或某些框架/控件内部的泵）提前取走并分发了消息。于是你在主循环里再也看不到那条消息，感觉像是“自己跑掉了”。结论是：遇到“消息到不了主循环”的现象，先排查进程/线程里是否存在第二个消息循环或任何会 PeekMessage + DispatchMessage 的路径，因为很可能是你或库代码提前把它分发了。
