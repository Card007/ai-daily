🥇 **callgraph analysis**

[callgraph analysis](https://jyn.dev/callgraph-analysis/) — jyn.dev · 05-03 00:00 · ⚙️ Engineering

> Building a custom Rust lint that understands who-calls-what requires more than pattern-matching AST nodes; it needs whole-program reasoning about function reachability. The post outlines how to extract a call graph from the compiler’s intermediate representations (e.g., traversing rustc HIR/MIR and resolving function/trait method calls) and then use that graph to flag undesirable dependencies (like a restricted API being reachable from a given module or entrypoint). It highlights practical pitfalls such as dynamic dispatch, trait method resolution, generics/monomorphization boundaries, and dealing with external crates where only summarized information may be available. The approach frames lints as static analyses: compute the graph once, cache it, and then query it from a lint pass to produce actionable diagnostics. The takeaway is that “serious” lints become much more powerful when you treat them like lightweight program analyses rather than syntax checks.

💡 **Why Read This**: Worth reading if you want to move beyond simple Clippy-style checks and learn how call-graph-driven analysis unlocks higher-signal, architecture-enforcing Rust lints.
